Biography

I am a software engineer at in the semiconductor division at Samsung Electronics. At KAIST, I researched modeling and simulation of complex semiconductor tools, and developed a new prediction model that is accurate and robust, with little extra computational complexity. At Samsung, I initially worked as a cloud/systems engineer on our big data analytics cloud platform. I now work as a software engineer and develop libraries for analyzing semiconductor data.

My current interests and stuff that I would like to learn are:

  • Data manipulation performance in Python/R
  • REST API design, streaming JSON data
  • Distributed machine learning applications
  • Kubernetes, and containerization of applications

Setting up Minikube with corporate proxy

Minikube is an easy way to get started with Kubernetes. It runs as a single-node Kubernetes cluster inside a VM, and includes almost all features of k8s, so you can try them out before installing them to an actual cluster. Supported addons include: Kubernetes dashboard EFK Registry Ingress and so on. (see https://github.com/kubernetes/minikube/blob/master/docs/addons.md) Minikube creates a VM that includes Kubernetes and a Docker daemon. To use containers inside this VM, you need to pass on the proxy settings to the docker daemon and install the certificate as well.
